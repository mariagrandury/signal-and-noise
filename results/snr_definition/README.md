# `snr_definition/` — SNR variants × decision accuracy on Apertus

This directory holds per-task SNR/DA tables and the SNR-vs-DA correlation
plots, generated by:

- [`multilingual/run_apertus_snr_variants.py`](../../multilingual/run_apertus_snr_variants.py) — writes `snr_variants_per_task.csv`
- [`multilingual/analyze_snr_variants.py`](../../multilingual/analyze_snr_variants.py) — reads the CSV, writes everything else

The 12 Apertus runs are `apertus-{175M,350M,600M,1B}-fwEdu{30,60,90}-fw…-seed1904`.
SNR is computed from per-mix last-5 ckpts; DA is decision accuracy (rank
agreement) over 3 mixes per (size or ckpt) pair.

## Files

| Path | What it is |
|---|---|
| `snr_variants_per_task.csv` | Master table (104 tasks × 279 cols). Per task: signal/noise/snr for each of 22 variants × 4 sizes (264 cols) + 3 size-DA cols (`decision_acc_size_<175M|350M|600M>`) + 12 ckpt-DA cols (`decision_acc_ckpt_<6000|18000|28000>_<size>`). |
| `da_size/snr_vs_decision_accuracy.png` | Variants (rows, ranked by mean Pearson r) × 3 DA-size cols. Each cell: SNR(small_size) vs DA(small_size@last → 1B@last). One color per panel. |
| `da_size/snr_vs_decision_accuracy_<lang>.png` | Same grid restricted to one language. Languages with too few non-NaN DA cells are skipped (currently de, fr, th). |
| `da_size/heatmap_pearson_r.png` | (variants × languages) Pearson r between log10(SNR) and DA-size, pooled across the 3 size cols. RdBu, anchored at ±1. |
| `da_ckpt/snr_vs_decision_accuracy.png` | Same shape as the size grid but cols are `ckpt 6000 → max`, `18000 → max`, `28000 → max`, and each panel pools all 4 sizes (color = size). |
| `da_ckpt/snr_vs_decision_accuracy_<lang>.png` | Per-language version of the above (ja also skipped here — too few non-NaN ckpt-DA cells). |
| `da_ckpt/heatmap_pearson_r.png` | (variants × languages) Pearson r vs DA-ckpt. |
| `variant_correlation_matrix.png` | (22 × 22) Pearson r between log10(SNR) values across variants, pooled over (task × size). Solid red blocks = redundant variants. |
| `da_size_vs_da_ckpt.png` | One dot per variant: x = mean r vs DA-size, y = mean r vs DA-ckpt. Diagonal = both DA definitions agree on which SNR is best. |

## Key insights

- **Several variants are algebraically equivalent at n_mixes = 3.** With only 3 mixes the dispersion-family signals (`dispersion`, `range`, `mpd`, `quartile_deviation`, `rms_deviation`) are linearly proportional, so their `log10(SNR)` correlate at r ≥ 0.999 and produce identical Pearson r against DA. Treat them as a single variant. Two more redundancy clusters in `variant_correlation_matrix.png`: {`iqr`, `rel_dispersion`, `rel_mpd`, `rel_std`} (r ≥ 0.998) and {`discrepancy`, `star_discrepancy`} (r = 0.959).
- **DA is 4-level discrete in this setup** (3 mixes ⇒ DA ∈ {0, ⅓, ⅔, 1}). All Pearson r values are computed over a quantized y-axis, so absolute r magnitudes are capped well below what 25-mix DataDecide reports. Treat the values as ordinal evidence, not ground-truth correlations.
- **Best global variants are all in the dispersion cluster.** Mean Pearson r across languages: DA-size top is `mpd`/`dispersion`/`range`/`quartile_deviation` ≈ +0.258. DA-ckpt top is `dist_std`/`dispersion_shifted`/`aad` ≈ +0.23. The dispersion family is a sound default for both DA definitions.
- **Best variant per language differs noticeably.** Highlights from the heatmaps:
  - `ru`: dispersion-cluster, r ≈ +0.68 (the strongest correlation in the whole dataset).
  - `eu`, `es`, `sw`, `hi`: `dispersion_shifted` or `dist_std`, r in [+0.42, +0.52].
  - `tr`: `mad` (r = +0.51) — outlier-robust spread does best.
  - `en`: weak correlations under DA-size (best `discrepancy` r = +0.15), much stronger under DA-ckpt (`mad` r = +0.42). For English, ckpt-DA is the more informative signal.
  - `zh`: weak/negative under DA-size (best variant r ≈ -0.02), only modestly positive under DA-ckpt (r ≈ +0.31). The ranking SNR predicts is unreliable for zh under either DA.
- **DA-size and DA-ckpt disagree on a few variants.** `star_discrepancy_shifted`, `mad`, `gini`, `tukey`, `dispersion_shifted`, `star_discrepancy` all have r ~0.08–0.12 higher under DA-ckpt than DA-size. Pick DA-ckpt when the goal is "stop training early without losing the right mix"; DA-size when extrapolating to a larger model.
- **`rel_std_snr` is no longer degenerate.** After the fix that broadcasts cross-mix std as `data_noise`, `snr_rel_std_<size>` spans roughly [0.06, 9] (was [0.97, 1.05]). It's now a reasonable variant; in this dataset it sits in the redundancy cluster with `iqr`/`rel_mpd`/`rel_dispersion`.
- **`projection_snr` and `tukey_snr` are mostly negatively correlated with DA** at every language and both DA defs — the depth-based formulations don't track decision accuracy here. Avoid for Apertus.
- **Per-language coverage is uneven.** de, fr, th have ≤4 valid (task, size) cells and are skipped from the per-language plots (ja additionally for DA-ckpt). Don't draw conclusions from those languages without evaluating more benchmarks first.
